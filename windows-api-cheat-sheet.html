ウィンドウ メッセージ

メッセージ駆動の良いところ

  • PostMessage 関数を用いて指定した処理を遅延実行できる。
  • 各ウィンドウを疎結合にできる。 ヘッダー ファイルに関数を書かずにすむ。

PostMessage 関数を用いて指定した処理を遅延実行できる。

メッセージを使用

ユーザーは新しくユーザー メッセージまたはアプリケーション メッセージを定義できる。 メッセージを定義する場合は既存のメッセージとカチ合わないようにすること。

ウィンドウ メッセージ
区分 範囲 名前 説明
最小 最大
システム 0 0x399 システムで使用するために予約されている。
ユーザー 0x400 0x7999 WM_USER プライベート ウィンドウ クラスで使用され、コントロールに固有のメッセージが定義されている。
  • コモン コントロールによって 0x700 付近のメッセージが使用されている。
  • ユーザーが定義したウィンドウ クラスについて、IsDialogMessage 関数を実行する場合は 0x400 付近のメッセージが使用される。
アプリケーション 0x8000 0xBFFF WM_APP Windows バージョン 4 以降としてマークしているアプリケーションで使用できるメッセージ。
0xC000 0xFFFF RegisterWindowMessage 関数を呼び出すアプリケーションで使用する文字列メッセージ。
システム 0x10000 システムによって予約されている。
