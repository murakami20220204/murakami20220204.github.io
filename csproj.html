分類 名前 説明 要素 備考
AutoGen <AutoGen> True </AutoGen>

Project <Project Sdk="Microsoft.NET.Sdk">
PropertyGroup <PropertyGroup Condition="'$(Configuration) | $(Platform)' == 'Debug | AnyCPU'">
アプリケーション全般 出力の種類 ビルドするアプリケーションの種類を指定します。 OutputType <OutputType> WinExe </OutputType>
ターゲット フレームワーク アプリケーションの対象となる .NET のバージョンを指定します。このオプションには、コンピューターにインストールされている .NET のバージョンに応じて異なる値を指定できます。 TargetFramework <TargetFramework> net7.0-windows </TargetFramework>
ターゲット OS このプロジェクトの対象になるオペレーティング システムのバージョンを指定します。
ターゲット OS バージョン
サポートされている OS バージョン プロジェクトが実行される最小 OS バージョンを指定します。指定しない場合、ターゲット OS のバージョン値が暗黙的に指定されます。以前のバージョンを使用する場合、それより新しいバージョンの API にガードを追加するにはコードが必要です。 SupportedOSPlatformVersion <SupportedOSPlatformVersion> 8.0 </SupportedOSPlatformVersion>
Windows フォーム このプロジェクトで Windows フォームを有効にします。 UseWindowsForms <UseWindowsForms> true </UseWindowsForms>
Windows Presentation Foundation このプロジェクトで WPF を有効にします。 UseWPF <UseWPF> True </UseWPF>
スタートアップ オブジェクト アプリケーションの読み込み時に呼び出されるエントリ ポイントを定義します。通常、これはアプリケーションのメイン フォームか、アプリケーションの起動時に実行する必要がある 'Main' プロシージャに設定されます。クラス ライブラリではエントリ ポイントが定義されていません。 StartupObject <StartupObject> WinFormsApp1.Program </StartupObject>
アセンブリ名 アセンブリ マニフェストを保持する出力ファイルの名前を指定します。 AssemblyName <AssemblyName> $(MSBuildProjectName) </AssemblyName>
既定の名前空間 プロジェクトに追加するファイルの基本名前空間を指定します。 RootNamespace <RootNamespace> $(MSBuildProjectName.Replace (" ", "_")) </RootNamespace>
アプリケーション Win32 リソース アイコン プログラム アイコンとして使用する .ico ファイルを設定します。 ApplicationIcon <ApplicationIcon> Icon1.ico </ApplicationIcon>
リソース アプリケーション リソースの管理方法が指定されます。
マニフェスト アプリケーションがユーザー アカウント制御 (UAC) を使用して Windows Vista またはそれ以降で実行されるときに、マニフェスト生成オプションを選択します。オプションには、既定のマニフェスト、マニフェストなし、プロジェクトによって提供されるカスタム マニフェストが含まれます。 NoWin32Manifest <NoWin32Manifest> true </NoWin32Manifest>
マニフェスト パス カスタム マニフェスト ファイルへのパスです。 ApplicationManifest <ApplicationManifest> App.manifest </ApplicationManifest>
アプリケーション パッケージ化 パブリッシュ "dotnet publish" を使用するなど、Visual Studio 外でのパブリッシュを許可します。 IsPublishable <IsPublishable> True </IsPublishable>
Global Usings 全般 暗黙的な global using プロジェクト SDK によって宣言される、暗黙的な global using を有効にします。 ImplicitUsings <ImplicitUsings> enable </ImplicitUsings>
暗黙的な global using を管理する 参照するプロジェクトの共通の global using ディレクティブを管理します。
ビルド全般 条件付きコンパイル シンボル 条件付きコンパイルを実行するためのシンボルを指定します。 DefineConstants <DefineConstants> $(DefineConstants); AAA </DefineConstants>
プラットフォーム ターゲット 出力ファイルの対象となるプロセッサを指定します。[すべての CPU] を選択すると、すべてのプロセッサが許容可能に指定され、アプリケーションを幅広いハードウェアで実行できるようになります。
Null 許容 プロジェクト全体での C# の null 許容コンテキストを指定します。C# 8.0 以降を使用するプロジェクトでのみ使用できます。 Nullable <Nullable> enable </Nullable>
アセンブリ情報の生成 ビルド中にプロジェクト プロパティをアセンブリ属性に変換します。
アンセーフ コード 'unsafe' キーワードを使用するコードをコンパイルできるようにします。
コードの最適化 より小さく、高速で効率的な出力に向けて、コンパイラの最適化を有効にします。
デバッグ シンボル ビルド中に生成されたデバッグ シンボルの種類を指定します。
ビルド エラーおよび警告 警告レベル コンパイラの警告で表示するレベルを指定します。レベルが高いほどより多くの警告が生成され、下位レベルのすべての警告が含まれます。
警告の抑制 1 つ以上の警告を生成するコンパイラの機能をブロックします。複数の警告番号はコンマ (',') またはセミコロン (';') を使用して区切ります。
警告をエラーとして扱う 警告をエラーとして扱うようコンパイラに指示します。
特定の警告をエラーとして扱う 指定された警告をエラーとして扱います。複数の警告番号はコンマ (',') またはセミコロン (';') を使用して区切ります。
ビルド出力 基本出力パス ビルド時のプロジェクトの出力の基本場所を指定します。プロジェクト構成を区別するために、サブフォルダーがこのパスに追加されます。
基本中間出力パス ビルド時のプロジェクトの中間出力の基本場所を指定します。プロジェクト構成を区別するために、サブフォルダーがこのパスに追加されます。
参照アセンブリ プロジェクトのパブリック API を含む参照アセンブリを生成します。
ドキュメント ファイル API ドキュメントを含むファイルを生成します。
ビルド イベント ビルド前のイベント ビルドの開始前に実行されるコマンドを指定します。プロジェクトが最新の場合は実行されません。終了コードを 0 以外にすると、ビルドは実行前に失敗します。
ビルド後のイベント ビルドの完了後に実行されるコマンドを指定します。ビルドに失敗した場合は実行されません。.bat ファイルを呼び出すには、 'call' を使用します。終了コードを 0 以外にすると、ビルドは失敗します。
ビルド後のイベントをいつ実行するか ビルド後のイベントが実行される条件を指定します。
ビルド厳密な名前付け アセンブリに署名する 出力アセンブリに署名して、厳密な名前を付けます。
ビルド詳細 言語バージョン このプロジェクトのコードで使用できる言語のバージョン。
算術オーバーフローのチェック 整数演算で範囲外の値が算出された場合に例外を発生させます。
決定論的 同一のコンパイル入力に対して同一のコンパイル出力を生成します。
内部コンパイラ エラーの報告 内部コンパイラ エラー (ICE) レポートを Microsoft に送信します。
ファイルの配置 出力ファイルのセクションを配置する位置をバイト単位で指定します。
パッケージ全般 ビルド時に NuGet パッケージを生成 ビルド操作中にパッケージ ファイルを生成します。 GeneratePackageOnBuild <GeneratePackageOnBuild> False </GeneratePackageOnBuild>
パッケージ ID nuget.org またはパッケージが存在するギャラリー内で一意である必要のある、大文字と小文字を区別しないパッケージ識別子。ID には、URL に無効なスペースや文字を含めることはできません。一般的に言って .NET 名前空間の規則に従います。 PackageId <PackageId> $(AssemblyName) </PackageId>
タイトル 一般的に UI で使用されるパッケージのわかりやすいタイトルは、nuget.org と Visual Studio のパッケージ マネージャーのように表示されます。 Title <Title> Title </Title>
パッケージ バージョン major.minor.patch のパターンに従うパッケージのバージョン。バージョン番号にはプレリリースのサフィックスが含まれる場合があります。 Version <Version> $(VersionPrefix) </Version>
作成者 nuget.org のプロファイル名に一致するパッケージ作成者のコンマ区切りリスト。これらは、nuget.org の NuGet ギャラリーに表示され、同じ作成者によるパッケージを相互参照するために使用されます。 Authors <Authors> $(AssemblyName) </Authors>
会社 Company <Company> $(Authors) </Company>
製品 Product <Product> $(AssemblyName) </Product>
説明 パッケージの UI 表示用の説明。 Description
著作権 パッケージの著作権の詳細。 Copyright
プロジェクトの URL UI 表示や nuget.org によく表示される、パッケージのホーム ページの URL。 PackageProjectUrl
アイコン パッケージのアイコン画像。画像ファイルのサイズは 1 MB に制限されています。サポートされているファイル形式には、JPEG と PNG が含まれます。128 ⨯ 128 の画像解像度をお勧めします。 PackageIcon
README パッケージの README ドキュメント。マークダウン (md) ファイルである必要があります。 PackageReadmeFile
リポジトリ URL パッケージのソース コードが存在するか、そのビルド元となるリポジトリの URL を指定します。プロジェクト ページにリンクするには、代わりに [プロジェクトの URL] フィールドを使用します。 RepositoryUrl
リポジトリの種類 リポジトリの種類を指定します。既定値は 'git' です。 RepositoryType
タグ パッケージについて説明するタグやキーワードをセミコロンで区切って並べたリスト。検索やフィルター処理でパッケージを見つけやすくするために使います。 PackageTags
リリース ノート このリリースのパッケージで行われた変更の説明。Visual Studio パッケージ マネージャーの [更新] タブなどの UI で、パッケージの説明の代わりによく利用されます。 PackageReleaseNotes
.NET ツールとしてパック このプロジェクトを、"dotnet tool" コマンドでインストールできるコンソール アプリケーションを含む特別なパッケージとしてパックします。
パッケージ出力パス パッケージを削除する出力パスを決定します。
アセンブリ ニュートラル言語
アセンブリ バージョン AssemblyVersion
ファイル バージョン FileVersion
パッケージ ライセンス パッケージ ライセンス プロジェクトのパッケージのライセンスを指定します。
プロジェクト シンボル シンボル パッケージを生成する プロジェクトがパックされたときに、追加のシンボル パッケージを作成します。
Code Analysis すべてのアナライザー ビルド時に実行 ビルド時は常にコード分析を実行します。
ライブ分析時に実行 入力中にエディターでコード分析を実行します。
Code Analysis .NET アナライザー ビルド時にコード スタイルを適用する ビルド時にコード スタイルに関する診断を作成します。
.NET アナライザーを有効にする .NET アナライザーを実行して API の使用をサポートします。
分析レベル プロジェクトで実行する必要のある一連のアナライザー。
